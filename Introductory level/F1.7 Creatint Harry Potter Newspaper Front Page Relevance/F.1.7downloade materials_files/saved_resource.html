Relevance

Today, we will summarize the material we have learned so far and work on an exciting project. It will be a newspaper with an animated image, like the newspapers in the Harry Potter movies. We will use everything we have learned in our previous lessons. Apart from that, we will also learn new properties: float, opacity, and overflow.

This project will prepare you to work with more complex templates and create more elaborate websites that consist of different details. In other words, we will build this newspaper, just like you can build something with Lego, using blocks.

Methodical material

Revision of the previous lessons

First of all, let’s revise the previous lessons. Especially the last one. What CSS properties did we use to set a background image to an element?

Information for the teacher: the background-image, background-size, background-position, background-attachment, background-repeat properties

Do you remember what the background-image property does?

Information for the teacher: it sets a background image to an element.

What is text-shadow?

Information for the teacher:it’s a property that can add a shadow to the text. Unlike box-shadow, it traces the outline of the text and not the frame of the tag container.

Why do we need the <hr> tag?

Information for the teacher: it creates a horizontal line.

Why do we use the import keyword in CSS? What does it do?

Information for the teacher: It allows you to connect the CSS files, also you can use it for importing external fonts.

What is the difference between margin and padding?

Information for the teacher: Margin creates some space around the element, and padding creates some space between the element's contents and its border.

Now you need to mark the “To revise the previous lesson” task as done by swapping the minus (-) sign for the plus (+) sign

Create an img folder (still inside the Daily Prophet folder) that will contain the images for the project. Create a template folder. We will put the newspaper template in the folder so that it will always be at our fingertips.

Styling the main container

The container has been created. Now it’s time to style it. Let’s open our style.css file and set some styles for our newspaper.

Offer the student to tell what styles should be applied to the newspaper. First of all, we will need to set the width of our newspaper because the screen has a landscape orientation and a newspaper has a portrait orientation. You can choose any value, but not a very large one. Otherwise, the newspaper will be stretched and disrupt the aspect ratio. It’s best to choose a value between 1000px and 1300px.

.daily-prophet{

width: 1300px;

} 

Let’s save and check it out. It seems like it didn’t change anything. Well, that is exactly what was supposed to happen because a block doesn’t have a height. Remind the student that it happened because of the block elements features. Their height depends on the contents of the block. Right now, there’s nothing inside the <div> that’s why its height is equal to zero. In the future, it will suit us, but right now, to test it, let’s set the height manually.

For now, you can set any height.

.daily-prophet{

width: 1300px;

height: 1200px;

} 

The newspaper has a special background color, the color of old paper.

Ask the student to get the color from the template using the browser and set the background color for the <div>

Oops... The same old problem again. The block is not centered!

Offer the student to remember the previous lesson, or he can peek into the code of that lesson and find the property we use to put a block in the middle. If the student is gifted, you may ask him to explain how the property works.

And if the student has failed to find it and is facing any difficulties, you can give him a hint: margin: auto. The auto value subtracts the width of the block from the parent element's width and divides the remaining space into two parts, setting them as right and left margins.

.daily-prophet{

width: 1300px;

height: 1200px;

background-color:#E7E2DA;

margin: auto;

}

Creating the Newspaper Heading

So, we have created the basic container. Now we need to fill it out. As always, we start creating the elements from the top. Please, tell me, what is the top section of the newspaper called? Correct, it’s called newspaper header. The font in the header is unique. We can’t find anything like this on the Internet. Also, there’s a coat of arms. That’s why we will use an easier way to add these things to our website. Instead of adding the text, we will create a <div> and set a background image.

Let’s add the tag and specify a class: class=”paper-header.”

<body>

<div class = "daily-prophet">

<div class="paper-header"></div>

</div>

</body> 

That’s it. No additional tags or text are needed. Now let’s open the style.css file and set the background image. Also, we need to set the height of the block. Why? For the same reason as before. If we don’t set the height, then it will be equal to zero. So, let’s set it. You can try out different values.

Information for the teacher: in general, 250px is a pretty good option but allow the student to try out different values and choose one independently.


background-position: center;

background-repeat: no-repeat;


If the student is proficient enough, you can tell him about what’s written below (about the shorthand background property).

As you might remember, in the previous lesson, we wrote each property separately, right?

Information for the teacher: here, you need to give the student an idea that some CSS properties have a shorthand variant. And the background property is one of them.

Tell the student that when there are many blocks with the background images on the website, it will be too difficult for the developer to specify all the properties separately. That’s why the CSS developers created shorthand properties. Let’s try it out.

Let’s remove the background-image property. Instead of it, let’s use the shorthand background property. Let’s specify url() (the image path) as a value. Then put whitespace and specify the position and the size, separated with the “/” symbol, then one more whitespace and add no-repeat. Only then insert a semicolon.

To make it easier for the student to figure out the order of values, follow this link https://developer.mozilla.org/ru/docs/Web/CSS/background or look up “background CSS” on the Internet and explain to the student the order of values in the background property and how to work with the learning resource. MDN’s benefits are that there are examples of using the properties, and everything is explained.

.paper-header{

height: 200px;

background-image: url("img/paper-head.PNG");

background: url("img/paper-head.PNG") center/cover no-repeat;

}

Creating a container that will store the newspaper articles

It’s getting more interesting. We have added the headings. Now we can work on the main content of the newspaper. First, we will create a common container. Why do we need that? It’s a good practice to create a separate parent container for the content of the page. And also, we will need to add borders. We will need this container for that, too.

Let’s create a <div> element. We need to do the same thing as before. In other words, we need to create a block inside our <div class = “daily-prophet”>. Remind about the class names. They must be sensible and give an idea about the content of the block. In this case we can name our class “paper-content”.

<body>

<div class = "daily-prophet">

<div class="paper-header"></div>

<div class="paper-sub-header"></div>

<div class="paper-content"></div>

</div>

</body>

Let’s style the block. Set a border for it.

.paper-content{

border:3px solid black;

} 

In the image below, our new block is circled. Ask the student why it’s flat.

The answer: there’s no content inside, meaning the height is equal to zero.

But that’s not all. We need to open the template. We can see that the newspaper in the template has padding, but there’s no padding on our webpage. Indeed, if we take a newspaper, a book, or even an article on the Internet, we can see some space between the contents and the edges of the page. Ask the student to remember which value can be used to do that.

The property we need is called padding. Why this property? Just because the space will be responsive. Even when we will add new blocks and articles to the newspaper, there still will be padding. If we set a margin for the inner elements, we must specify it for each new element. It’s pretty inconvenient.

Set the padding for the main container of the newspaper.

daily-prophet{

width: 1300px;

height: 1200px;

background-color:#E7E2DA;

margin: auto;

padding: 50px;

} 

Here’s the result.

Creating an Article Block

So, it’s time to add our first article. First, we need to create a structure for it. Since we use a box layout, let’s create a container block. Please, remind me of the name of the container tag. It’s a <div>. Let’s add it. Offer the student to think about where to add this <div>. It would be best to write it inside our content block.

Let’s start. Create a <div> with the “first-article” class.

<body>

<div class = "daily-prophet">

<div class="paper-header"></div>

<div class="paper-sub-header"></div>

<div class="paper-content">

<div class = "first-article"></div>

</div>

</div>

</body> 

The article will consist of 4 elements: a heading, a divider, an image, and text.

Offer the student to write the tags on his own. Help him if needed.

<h1> <hr> <img> <p>.

Now it’s time to talk about the <img> tag’s alt=”” attribute. We need to specify and fill it for each image. This attribute is used for displaying some text if the image fails to load but also to help the search engines, such as Google and Yahoo, find your website at the users’ request.

Because the search engines cannot see the images, but they can read the alt attribute because it’s presented in the form of text. But the main reason why you should specify this tag is to make it easier for visually impaired people to use the website. The thing is that there’s a program called Screen Reader it reads the text written on the website out loud. If you don’t specify the alt attribute for the image, Screen Reader will just say “Image,” but it won’t be able to tell anything about the image itself. That’s why if you specify the alt attribute, it will allow the program to read the image description out loud. The description must be succinct.

Information for the teacher: in the example, the alt attribute contains the description of the image in the lesson materials folder. The student can write his own image description, if he knows what image he will choose. But don’t insert the image path in the src attribute just yet.

<div class = "daily-prophet">

<div class="paper-header"></div>

<div class="paper-content">

<div class = "first-article">

<h1></h1>

<hr>

<img src="" alt="Harry Potter eating some candy">

<p></p>

</div>

</div>

</div>

</body>

Information for the teacher: now, you can ask the student to save the file and check how the image is displayed without the file path.

Don’t forget about the classes. Because we will add articles in the future, tags might coincide and the styles will work incorrectly.

<body>

<div class = "daily-prophet">

<div class="paper-content">

<div class = "first-article">

<h1 class="first-article-title"></h1>

<hr>

<img class="first-article-img" src="" alt="Harry Potter eating some candy">

</div>

</div>

</div>

</body>

Float and Opacity Using the Example of an Article

Great, everything works! And the gif has shown up. But we have two problems:

1. The image is too vivid for a newspaper.

2. The text and image are not placed like in a real newspaper.

Let’s fix it and style the whole block. First of all, as usual, let’s write all the necessary class names in the style.css file.

.first-article {

}

}

} 

Let’s start with the heading. It must be aligned to the center and enlarged. Ask the student to do it on his own. If he can’t do it, help him - text-align: center, font-size. He can choose any font size.

}

.first-article-title {

text-align: center;

font-size: 100px;

}

.first-article-img {

}

.first-article-text {

} 

The image must be dimmed. You need to make it look like it was printed in an actual newspaper. That’s why we make it more transparent.

The property is called opacity. It can change the opacity of the HTML blocks and their content. The values are set in the range from 0 to 1. 0 means completely transparent, 1 is completely opaque. Note that if the block becomes fully transparent, it will become invisible but still occupy its space. It won’t vanish entirely. Let’s try.

Set the opacity for the image to 0 and check it out with your student. The image will be invisible, but the text won’t move up because the image will still be occupying its own space.

We need a value around 0.7-0.8.

.first-article {

}

.first-article-title {

text-align: center;

font-size: 100px;

}

.first-article-img {

opacity: 0.7;

}

.first-article-text {

} 

And now the next property. The Float is a special property. It aligns an object to the left or the right, but the adjacent elements will wrap around it. Since the property aligns an object, its possible values are left and right. Let’s try it out. Set float:left / float:right to the image.

Note that the float must be set to the image, and the text will automatically adjust to it.

.first-article {

}

.first-article-title {

text-align: center;

font-size: 100px;

}

.first-article-img {

opacity: 0.7;

float: right;

}

.first-article-text {

} 

Let’s check.

Let’s decorate it all a bit, add padding to the article container and the left and bottom margin to the image to create some space between the image and the text.

Setting The Fonts and The Height of The Line

The font we use on the page now doesn’t look right. That’s why we will use the Google Fonts website to add a font. But before we do that, please, tell me if you remember what property is used to change the font on the webpage. It’s font-family. And what keyword helps to load the font on the page? Import.

Go to Google Fonts and select a font. The student can choose any font he wants. The fonts used in the project are Berkshire Swash (heading), Henny Penny(text).

Open the Google Fonts website, pick a font and click on select this style

A window will pop up on the right. Click on import, copy everything but the <style> tags from the first text field. Ask the student to think about why we are not copying the <style> tag.

We don’t copy it because we write our styles in a separate CSS file and import our font there. And the <style> tag is needed only in HTML.

Copy the content of the second text field (font-family). You need to add it to the elements of the font of which you want to change.

Add the @import statement on the first line of the style.css file.

And the font-family rule must be added where it’s needed.

@import

url('https://fonts.googleapis.com/css2?family=Henny+Penny&display=swap');

@import

url('https://fonts.googleapis.com/css2?family=Berkshire+Swash&display=swap');

}

}

opacity: 0.7;

float: right;

margin-left: 20px;

margin-bottom: 20px;

}

} 

Great, but it seems like the lines in our article don’t have enough space between them. Let’s add some more space. We will use a special property called line-height for this. It allows setting the height of each line of the text. The value can be specified in different measurement units, for example, in pixels. Let’s try specifying it in pixels.

The values are up to the student. The value used in the example is 32px.

padding: 30px;

}

.first-article-title {

text-align: center;

font-size: 100px;

}

opacity: 0.7;

float: right;

margin-left: 20px;

margin-bottom: 20px;

}

.first-article-text {

font-family: 'Henny Penny', cursive;

line-height: 32px;

} 

The final result.

Adding margin to the content section of the newspaper, removing the main block height

We need to fix a couple of things. The first one is the space between the header section and the content section of the newspaper. The second one is to remove the main newspaper container height.

As you might remember, at the beginning of the lesson, we set a height to the newspaper container <div>. Why did we do it? To see the block itself, because as long as it doesn’t have any content inside, its height equals zero. But a block adjusts to the size of the content unless the height is specified. That’s what we need now. Let’s remove the height from the daily-prophet.

.daily-prophet{

width: 1300px;

height: 1200px;

background-color:#E7E2DA;

margin: auto;

padding: 50px;

} 

And now, let’s divide our header section from the content section. What property do we need?

The Margin – because we need to create some space around the elements. We need to add some space between the borders of our content block and the <div>s at the top of the page.

.paper-content{

margin-top: 30px;

border:3px solid black;

}

But as you can see, a problem remains. It’s easy to fix it. We just need to add a property called overflow. This property allows us to show or hide the scroll bar if the element’s content is too big to fit. As you might have noticed, the problem occurred only after we specified the float property because all elements have an overflow property with the value of visible by default, but unfortunately, this value sometimes comes into conflict with float. As in our example. In this case, we have two possible solutions:

To increase the height of the parent block manually;

To specify overflow: hidden for it;

The hidden value will crop the content inside the block if the size of the content is bigger than the size of the block. In other words, it cancels out the standard behavior when we have a scroll bar if the content is too big. But in this case, it allows to avoid the property conflict and let the block expand to fit the content automatically. That’s why we will use it.

The parent block in this case is the <div> that we created as a container for our articles. Let’s write the property for it.

.paper-content{

margin-top: 30px;

border:3px solid black;

overflow: hidden;

}
